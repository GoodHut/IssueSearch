 URL : "https://github.com/ilkarman/DeepLearningFrameworks/issues/2" TITLE : quick performance note BODY : hi - thanks for setting up the benchmark. i want to quickly put a performance note so folks are not surprised when seeing the perf numbers. first, if you measure speed difference for basic ones like cnns, something is wrong in how you use the frameworks : in this case, the major performance difference comes from i/o, not the framework itself. if you look at the tensorflow and caffe2 examples, the data is provided with a feed approach - this is usually bad for performance, and instead you should use a db or input iterator to do so. under the hood, this makes prefetching and other optimizations possible and is particularly important for performance.
