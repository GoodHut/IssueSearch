 URL : "https://github.com/leoallvez/Casa/issues/2" TITLE : melhorias na segurança BODY : contexto: o sistema casa http://casa-sistema.com.br permite o cadastro de várias instituições, porém a base de dados é a mesma para todas instituições cadastradas no sistema. a forma usada para se saber de onde é um determinado registro é a chave estrangeira instuicao_id presente em boa parte das tabelas do banco de dados. o problema: pelo fato de compartilha a base de dados com todas instituições, é possível ter acesso de informações de outras instituições apenas sabendo o id do cadastro. exemplo : sou um usuário do sistema x realiza o cadastro um adotivo um criança para adoção , esse adotivo recebe o id 27. se por um acaso o usuário z de outra instituição acessar o link: http://casa-sistema.com.br/adotivos/27/edit editar e salva ele terá alterado o cadastro de outra instituição que não é dele. existem situações que o usuário também pode ter acesso a informações referente a outro nível de acesso. o que foi feito: até o presente momento nenhuma providência foi tomada para a correção desse problema, o único controle que temos é qual o nível de acesso do usuário e de qual instituição ele pertence. como solucionar: o sistema só deve permitir que o usuário veja as informações da sua instituição e seu nível de acesso. na documentação do laravel é possível ver algumas possíveis soluções, porém não fiz um estudo profundo de qual seria a melhor: link https://laravel.com/docs/5.3/authorization
