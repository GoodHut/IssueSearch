 URL : "https://github.com/potfit/potfit/issues/17" TITLE : compiling using mkl BODY : so, when compiling with mkl i find that there's a library issue in the makefile.inc that can be fixed if every line that looks like so: cinclude += -i${mkldir}/include libs += -wl,--start-group -lmkl_intel_lp64 -lmkl_sequential \ -lmkl_core -wl,--end-group -lpthread instead were: cinclude += -i${mkldir}/include libs += -l${mkldir}/lib/intel64 libs += -wl,--start-group -lmkl_intel_lp64 -lmkl_sequential \ -lmkl_core -wl,--end-group -lpthread essentially, the make cannot find the mkl libraries/includes. this could in theory just be user error that being the libraries weren't installed properly on the user's side , but it's an easy fix that makes installation more robust and error-free. henry
