 URL : "https://github.com/gilmoreorless/vscode-zoneinfo/issues/1" TITLE : ideas for features BODY : this was created as a direct port of the sublime-zoneinfo gilmoreorless/sublime-zoneinfo package for syntax highlighting. however, the vscode plugin infrastructure allows more possibilities for navigating the tz source files. given the niche interest area, i'm under no illusion that this extension will be widely used, so i'm just writing down some ideas for future features rather than jumping into implementation straight away. it's important to keep in mind that the tz files are not changed very often, compared to how often they're viewed, so features should focus on viewing and navigating rather than authoring. symbol definitions for zone and rule sections, which enables more features: quick navigation with go to symbol https://code.visualstudio.com/docs/editor/editingevolved _go-to-symbol go to definition https://code.visualstudio.com/docs/editor/editingevolved _go-to-definition and click-through support, especially for zone -> rule and zone -> zone links definition peeking https://code.visualstudio.com/docs/editor/editingevolved _peek _ other ideas are welcome _
