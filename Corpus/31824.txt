 URL : "https://github.com/liam2/larray/issues/149" TITLE : swap axis arguments BODY : it would be convenient to be able to create axes without name like this: axis 'a,b,c' and axes with a name like this: axis 'a,b,c', 'name' and axis 'name=a,b,c' however, this is awfully backward incompatible. the least disruptive way to go forward that i see is to change the signature of axis like this: axis oldname=none, labels=none, name=none and output a deprecation warning if oldname is used saying people should use the transitional syntax: axis labels=labels, name=name then, in the next release, switch the warning to an error. and after a couple release, we could finally change the signature to: axis labels, name=none
