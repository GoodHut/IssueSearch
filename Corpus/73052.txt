 URL : "https://github.com/pciet/wichess/issues/8" TITLE : combat automated clients BODY : playing the game with automated clients is an easy way to gain a number of the special pieces and/or is a good programming challenge, but loading the server with relentless requests is not ideal. testing the max-speed automated client on a local machine also bogs down the browser, so it is not clear that way what the user experience looks like with server requests being delayed. one solution may be to require a captcha to start each game and to login, or even per-move if moves are happening too quickly. another may be to require a delay of 100+ms between each request, calculated by the server.
