 URL : "https://github.com/patternfly/patternfly-design/issues/500" TITLE : need icon enahancements BODY : as i understand it, there is currently a very limited set of icons that are known to work well with patternfly. https://www.patternfly.org/styles/icons/ keycloak needs to allow administrators to select and/or upload icons that will eventually be displayed in patternfly templates. the very limited icon set and even more limited color use will make for a poor user experience. the use case for this is that a keycloak administrator would assign an icon to each application managed by keycloak. these icons would show up on the sso login screen and in the account management console. if you want an idea of what i'm talking about, check out google's login screen where they show an icon for every app that you can have access to: https://ps.w.org/google-apps-login/assets/screenshot-2.png?rev=1629984 at the very least, a detailed tutorial on how to create patternfly-compatible icons would be extremely helpful. to make this really shine, patternfly needs to allow more icons than it currently accepts.
