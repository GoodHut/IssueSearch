 URL : "https://github.com/newvem/pytz/issues/7" TITLE : incorrect cet time. BODY : i'm developing a reporting tool that works on cet. for this i localize my dates using the pytz.timezone 'cet' . however this automatically switches to cest when my date is in the dst region, whilst the cet timezone is not supposed to do this. in my opinion this should not happen. for now i'll switch to utc+1 instead of cet. example code: setup: from datetime import datetime import pytz tz = pytz.timezone 'cet' first example: date = datetime 2000, 1, 1 loc_date = tz.localize date returns desired and expected : datetime.datetime 2000, 1, 1, 0, 0, tzinfo=<dsttzinfo 'cet' cet+1:00:00 std> second example: date = datetime 2000, 6, 1 loc_date = tz.localize date returns unfortunately datetime.datetime 2000, 6, 1, 0, 0, tzinfo=<dsttzinfo 'cet' cest+2:00:00 dst>
