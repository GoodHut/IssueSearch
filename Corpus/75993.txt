 URL : "https://github.com/UofSSpaceTeam/usst-navigation/issues/2" TITLE : improve image processing BODY : we should take a look at the current computer vision code and meassure its performance on raspberry pi's, and think of ways to improve performance. i was thinking it might be possible to create a pipelined architecture where multiple pi's are used, one for each stage in the image processing. theoretically if there are three stages to the processing and each stage is done on a different machine, the frame rate could be increased only to be limited to the slowest operation in the pipeline. the catch is that it also takes time to send images from one machine to another, so the performance gain could be negated.
