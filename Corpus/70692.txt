 URL : "https://github.com/slashdotdash/eventstore/issues/55" TITLE : writer per event stream BODY : currently event store uses a single writer process eventstore.writer https://github.com/slashdotdash/eventstore/blob/cb83101db3c6c4f60c475f24a79115a61bb1b57b/lib/event_store/writer.ex to append events to postgresql. the writer assigns the monotonically incrementing event id during insert. to support running on a cluster of nodes 53 , we need to support multiple writers, distributed amongst the cluster: one writer per event stream. this means that event id generation must also move to postgresql. it's possible to use a function to assign a value in postgresql e.g. event_id from a value in another table during insert. this is to achieve gapless event ids which are not guaranteed when using a sequence . - gapless sequences for primary keys http://www.varlena.com/generalbits/130.php - gapless sequence without locking the whole table http://mssql-to-postgresql.blogspot.co.uk/2007/12/gapless-sequence-without-locking-whole.html?m=1
