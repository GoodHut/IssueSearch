 URL : "https://github.com/LiskHQ/lisk/issues/1091" TITLE : avoid test index files BODY : we have run into the problem of forgetting to add a new test file on its corresponding index file. then we execute the suite, tests pass but our new test was never executed. to avoid that situation, i propose to change the mechanism: - instead of having an index file, scanning the target folder, his nested folders and finding all the .js files to execute them.
