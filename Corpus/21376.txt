 URL : "https://github.com/kamaln7/ansible-swapfile/issues/18" TITLE : falling back to dd BODY : either a bug or a warning, {{ swapfile_size }} having the units in the example doesn't work if you fall-back to dd, as the corresponding count parameter for dd should just be an integer. if you give it units, it will obediently convert them back to bytes first. worked example with swapfile_size=512mb swapfile_use_dd=false -- fallocate -l {{ swapfile_size }} {{ swapfile_location }} creates={{ swapfile_location }} will create a 512mb swapfile, but swapfile_size=512mb swapfile_use_dd=true -- dd if=/dev/zero of={{ swapfile_location }} bs=1m count={{ swapfile_size }} creates={{ swapfile_location }} will create a 512000000mb or 512 terabyte swap file, which for my purposes is a little excessive.
