 URL : "https://github.com/hcilab/processing-games/issues/84" TITLE : reduce size of repository BODY : the repository is getting to be very large, resulting in long clone times. here are a couple of steps we can take to reduce the size of the repo: convert .wav to .mp3 most of the audio is currently store stored as .wav files because there was an issue with the previous sound library which wouldn't loop .mp3 files. - ensure that new sound library supports looping of .mp3 files - convert all .wav files to .mp3 bfg repo-cleaner https://rtyley.github.io/bfg-repo-cleaner/ this tool will delete large files e.g., music, images that are no longer used by the current state of the project, preserving the ones that are still needed. usage: bfg --strip-blobs-bigger-than 50m --protect-blobs-from master,development my-repo.git
