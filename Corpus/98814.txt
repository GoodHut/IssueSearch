 URL : "https://github.com/rpav/cl-autowrap/issues/75" TITLE : libffi shared library name BODY : when loading cl-autowrap/libffi, i get an error about not being able to open the shared library libffi latest libffi by default will build out as libffi-6 if you see cffi's libffi loading, their form looks like this: define-foreign-library libffi :darwin :or libffi.dylib libffi32.dylib /usr/lib/libffi.dylib :solaris :or /usr/lib/amd64/libffi.so /usr/lib/libffi.so :openbsd libffi.so :unix :or libffi.so.6 libffi32.so.6 libffi.so.5 libffi32.so.5 :windows :or libffi-6.dll libffi-5.dll libffi.dll t :default libffi see here https://github.com/cffi/cffi/blob/master/libffi/libffi.lisp autowrap's looks like this: cffi:define-foreign-library libffi :darwin :or :framework libffi :default libffi :unix libffi.so :windows libffi.dll t :default libffi cffi:use-foreign-library libffi i think autowrap should use the same form cffi does, for completeness
