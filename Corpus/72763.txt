 URL : "https://github.com/odlgroup/odl/issues/1077" TITLE : chambolle pock algorithm BODY : i had a look at the implementation of the chambolle pock algorithm + its usage and i have a few thoughts to it. first, consider renaming it to pdhg primal-dual hybrid gradient . second, the parameter gamma is misleading. it should be used when f^ast is _strongly convex_ documentation says uniformly convex . if g is strongly convex, one can do the same but scale tau and sigma differently. the current implementation would scale them in both cases the same which is not correct . third, the gamma has been used for the tgv example which is not strongly convex but only what is so called partially strongly convex , see https://arxiv.org/pdf/1609.07373 for more details.
