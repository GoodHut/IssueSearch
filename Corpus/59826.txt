 URL : "https://github.com/CSC-IT-Center-for-Science/notebook-images/issues/7" TITLE : user 1001 -> user 1000 BODY : a strange docker directive of > user 1001 when no such user exists in the /etc/passwd of the containers seems to break pyspark. setting that to user 1000 jovyan, the default user seems to at least let pyspark load it's context. a side effect of this is that the user sees the username i-have-no-name and whoami says it doesn't know user number 1001. after the dust settles and this has been tested to work, we could consider moving all of them to user 1000.
