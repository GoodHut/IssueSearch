 URL : "https://github.com/FannieGirl/new-repo/issues/4" TITLE : 优雅使用 css modules BODY : css modules 很容易学，因为它的规则少，同时又非常有用，可以保证某个组件的样式，不会影响到其他组件。 看到这个段话我就很兴奋啊！如果对less sass 应用的不够熟悉的话，不妨学习一下这个。 我主要跟着阮一峰老师的教程学的。写的很棒！我在这里只是根据自己学的总结一下。 css的规则都是全局的，任何一个组件的样式规则，都对整个页面有效。 主要有一下几点： （一）、局部作用域：产生局部作用域的唯一方法，就是使用一个独一无二的class的名字，不会与其他选择器重名。 css modules 还提供一种显式的局部作用域语法:local .classname ，等同于.classname。 ! image https://cloud.githubusercontent.com/assets/19258229/24032083/2b5adb48-0b21-11e7-9a9f-1372ce1e8c6e.png （二）、全局作用域：:global .classname 的语法，声明一个全局规则。 ! image https://cloud.githubusercontent.com/assets/19258229/24032044/f18e080e-0b20-11e7-867c-0e5a450ed9b7.png （三）、定制哈希类名：css-loader默认的哈希算法是 hash:base64 ，这会将.title成._3zyde4l1yatcokgn-dbwel这样的字符串。 webpack.config.js里面可以定制哈希字符串的格式。 ! image https://cloud.githubusercontent.com/assets/19258229/24032136/60547408-0b21-11e7-98a2-1d14c1dbdf8f.png （四）、class 的组合：在 css modules 中，一个选择器可以继承另一个选择器的规则，这称为 组合 （ composition ）。 ! image https://cloud.githubusercontent.com/assets/19258229/24032175/90d79ec0-0b21-11e7-81c5-61c457d9a9f3.png （五）、输入其他模块：选择器也可以继承其他css文件里面的规则。 ! image https://cloud.githubusercontent.com/assets/19258229/24032213/c6255a22-0b21-11e7-87d5-b92e7711bdb1.png （六）、输入变量：css modules 支持使用变量，不过需要安装 postcss 和 postcss-modules-values。 阮一峰老师的教程很详细，又有demo。需要学习的移步去哪里学习吧！我主要是搬运过来，自己总结一下知识。 后言： 参考链接：http://www.ruanyifeng.com/blog/2016/06/css_modules.html
