 URL : "https://github.com/consul/consul/issues/1644" TITLE : absolute url on pagination BODY : consul use kaminari gem https://github.com/kaminari/kaminari to pagination. now in views with pagination the links generated are relative urls example on /proposals : <ul class= pagination text-center margin-top aria-label= pagination > <li> <a href= /proposals >first</a> </li> <li class= pagination-previous > <a rel= prev href= /proposals >previous</a> </li> <li> <a rel= prev href= /proposals >1</a> </li> <li class= current > <span class= show-for-sr >you are on page</span> 2 </li> <li> <a rel= next href= /proposals?page=3 >3</a> </li> <li class= pagination-next > <a rel= next href= /proposals?page=3 >next</a> </li> <li> <a href= /proposals?page=3 >last</a> </li> </ul> it seems kaminari doesn't have this setting and no plans to change it https://github.com/kaminari/kaminari/pull/822 to improve seo we need to use absolute urls on pagination . something like: <li> <a rel= next href= http://localhost:3000/proposals?page=3 >3</a> </li>
