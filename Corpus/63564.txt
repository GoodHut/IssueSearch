 URL : "https://github.com/links-lang/links/issues/183" TITLE : empty route unreachable BODY : since 157, on doing this: addroute ,foo servepages it no longer seems to be possible to get the result of foo by going to http://localhost:8080 or http://localhost:8080/ . it used to be i think that if you used the trailing slash the right thing would happen, but now either case results in nope . this is the case even if there is an index.html file in the current directory; though from the discussion in 157 i thought that the right behavior might be for index.html to be served if no other route matches. on the other hand, doing addroute foo ,foo servepages does cause the right thing to happen if you visit http://localhost:8080/foo .
