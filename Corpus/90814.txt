 URL : "https://github.com/harharkh/mayda/issues/3" TITLE : streaming decode / encode BODY : as an alternate to the current buffered decode and decode_into functions, the api could provide an iter of the encoded values, which could then be mapped, filtered, etc by the caller without need for allocating / filling an intermediary vec. since decoded integers would be passed over the stack, there could be some speed gains to be made in overall application performance. also, in some cases where only part of the data is required, the iteration could be abandonned early by the caller, reducing waste. conversely, an unbuffered appending encode api could make some patterns easier to implement and more efficient.
