 URL : "https://github.com/elyctech/ren2/issues/3" TITLE : decide on layering support BODY : layering is an important part of 2d rendering. however, to effectively do rendering in webgl, z values need to be arranged between 1 and -1, with negative numbers being on top of positive numbers. given this, z values must be generated based on the number of layers there are, and if the number of layers is allowed to change, the z values must be updated to reflect that. currently, all that is supported is providing a number denoting the layer. if that number falls outside of the -1 to 1 range, the layer is simply not shown. should the user be responsible for meeting the range requirements manually, or should ren2 provide full-support for managing z values?
