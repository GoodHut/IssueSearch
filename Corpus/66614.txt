 URL : "https://github.com/ThePhD/sol2/issues/362" TITLE : neat constructor syntax BODY : the syntax i picked for constructors sucks: cpp lua.new_usertype<thing> thing , // vomiting noises sol::constructors<sol::types<std::string, int>, sol::types<bool>, sol::types<>> , ... ; cpp lua.new_usertype<thing> thing , // happy smiley noises sol::constructors<void string, int , int bool , void > ... ; the second syntax is better. note that the return type of those function types should be ignored, as its only the arguments that matter a constructor will always make a thing , no matter what, so the return type shouldn't matter... but, it might be good to demand it be like that anyhow .
