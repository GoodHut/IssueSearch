 URL : "https://github.com/crowdcst/webpack-bundle-summary/issues/1" TITLE : filter files by extension BODY : problem: by default all assets in the bundle will be output to the summary file it's very common that when keeping track of bundle size you might want to only track certain files like javascript proposed solution: optional option filter which will take the asset name and return a boolean for weather or not this asset gets included in the summary example for matching only javascript assets: javascript new bundlesummary { filter: assetname => { return /\.js$/.test assetname } }
