 URL : "https://github.com/Theano/Theano/issues/6142" TITLE : slicing within tensordot BODY : hi, i noticed a weird result using slicing within tensordot. the following code shows how slicing in a call to theano.tensor.tensordot produces a different result from slicing in a call to a function mapping inputs to the result of theano.tensor.tensortdot. the second value seems to be right since numpy.tensordot produces the same result. i checked the t ..., 0 syntax t being a theano tensor which seems to work properly. import numpy as np import theano import theano.tensor as t floatx = theano.config.floatx a1 = t.matrix dtype=floatx b1 = t.tensor3 dtype=floatx c1 = t.tensordot a1, b1 ..., 0 testfunc1 = theano.function a1, b1 , c1 a2 = t.matrix dtype=floatx b2 = t.matrix dtype=floatx c2 = t.tensordot a2, b2 testfunc2 = theano.function a2, b2 , c2 mat = np.random.random 3, 3 .astype floatx tensor = np.random.random 3, 3, 2 .astype floatx print testfunc1 mat, tensor slicing in call to theano.tensor.tensordot print testfunc2 mat, tensor ..., 0 slicing in call to theano function print np.tensordot mat, tensor ..., 0 slicing in call to numpy.tensordot print np.tensordot mat, tensor 0 no slicing with numpy.tensordot this result was obtained using theano 0.9
