 URL : "https://github.com/metabase/metabase/issues/6385" TITLE : browser sockets limit BODY : - your browser and the version: chrome 58.0.3029.110 - your operating system: manjaro 17.0.6 - your databases: postgres and bigquery - metabase version: 0.26.2 - metabase hosting environment: docker - metabase internal database: postgres hi there. well, i'm not sure how this will be categorized. here, at contaazul https://contaazul.com , we have a dashboard with more than 30 cards. it is currently one of our most important dashboards. the cards are querying data from a postgres database aws rds . as it started to grow up and became slower over time. it takes approximately 15 minutes+ to fully load the dashboard. by doing some research i found out that browsers limits connections per hostname https://stackoverflow.com/questions/985431/max-parallel-http-connections-in-a-browser . chrome has a 6 connections limit apparently. i've recorded a video https://www.youtube.com/watch?v=eqwgfn85m4a showing how this makes things slower. the first dashboard is using postgres and the second one is querying bigquery. one big problem is that if a dashboard is loading, the user can't open another tab to do something else load another dash or make a new question while the dashboard loads, because the limit is per hostname. there are some ways to get around this http://blog.teamtreehouse.com/making-the-web-faster-with-spdy and http pipelining https://en.wikipedia.org/wiki/http_pipelining . i'm not sure if this really is the problem and i was thinking if enabling https could possibly solve this but i don't know if metabase supports the spdy protocol or if it would be any different.
