 URL : "https://github.com/XeroAPI/Xero-Net/issues/292" TITLE : zero based enums BODY : unless the upstream api requires it, please can we refrain from having valid 0 value enums such as account in objectgroup . it makes quite difficult to work out whether the bug is because you're using the wrong value, or whether you just haven't set a value in the first place this is especially highlighted when a new value is added upstream because, without a valid enum to translate it into until the wrapper gets updated it will just resolve to a 0
