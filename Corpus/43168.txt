 URL : "https://github.com/froala/wysiwyg-editor/issues/2346" TITLE : color picker behavior BODY : expected behavior. selecting a color in the color picker should fill the textbox with the hex value of that color. actual behavior. when you select a color, now that there is an ok button, the user expects to click that in order to set the color, despite the fact that the color changes immediately. if you select a color, then click ok, the dialog assumes you meant the empty textbox value for the color, and removes the color. steps to reproduce the problem. set a background color of a table cell, click ok. os. windows 7 pro browser. firefox latest , chrome latest
