 URL : "https://github.com/awslabs/s2n/issues/633" TITLE : fips + mutual auth support BODY : right now s2n supports either fips mode, or mutual auth mode, but not both at the same time. we will need to check the following when offering mutual auth to ensure we remain fips compliant: 1. ciphersuite 2. tls version 3. client cert sig/hash algorithm links: - https://www.openssl.org/docs/fipsnotes.html - https://wiki.openssl.org/index.php/fips_mode_and_tls - https://csrc.nist.gov/csrc/media/publications/fips/140/2/final/documents/fips1402.pdf - https://csrc.nist.gov/csrc/media/projects/cryptographic-module-validation-program/documents/fips140-2/fips1402ig.pdf
