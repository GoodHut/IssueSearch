 URL : "https://github.com/fusionpbx/fusionpbx/issues/2430" TITLE : call flow and say function BODY : if use say: function as a sound in call flow app, it's counted as a filename within playback function. this small hack resolves this. just change play_file function in call_flow.lua script. call_flow.lua --! @todo move to library // added possibility to use say local function play_file dbh, domain_name, domain_uuid, file_name if file_name:match say: then local say_string = file_name:gsub 'say: ','' log.noticef 'using say function with '..say_string..' parameters' session:execute 'say', say_string else local full_path, is_base64 = find_file dbh, domain_name, domain_uuid, file_name if not full_path then log.warningf 'can not find audio file: %s. try using it in raw mode.', file_name full_path = file_name else log.noticef 'found %s as %s %s', file_name, full_path, is_base64 and ' base64 ' or '' end session:execute playback , full_path ; end end
