 URL : "https://github.com/whosonfirst-data/whosonfirst-data/issues/793" TITLE : upgrade hosted libpostal BODY : this really only applies to venues but there are so many venue repos that this seemed a better fit. i noticed in the generated venues bundles that the addr: fields are being generated by calling libpostal with the concatenation of sg: and presumably other fields. for the address 1962 vandolah road po box 1348 wauchula fl 33873 , the hosted libpostal output https://libpostal.mapzen.com/parse?address=1962%20vandolah%20road%20po%20box%201348%20wauchula%20fl%2033873 is: json { label : house_number , value : 1962 }, { label : road , value : vandolah road po }, { label : house_number , value : box }, { label : postcode , value : 1348 }, { label : city , value : wauchula }, { label : state , value : fl }, { label : postcode , value : 33873 } resulting in addr: properties in the bundle file: json { addr:full : 1962 vandolah road po box 1348 wauchula fl 33873 , addr:housenumber : 1962;box , addr:postcode : 1348;33873 , addr:street : vandolah road po } the latest libpostal parses this correctly: > 1962 vandolah road po box 1348 wauchula fl 33873 result: { house_number : 1962 , road : vandolah road , po_box : po box 1348 , city : wauchula , state : fl , postcode : 33873 }
