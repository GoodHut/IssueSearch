 URL : "https://github.com/xnau/list_field/issues/4" TITLE : managing the database BODY : it would be tempting to set up the list field database the same way the main table is set up, but that may prove to be complex in terms of managing the columns and datatypes. the alternative how it is implemented in the current list field class i created for the member payments plugin is to set up the columns as general parameters: - column id indexes to the field definition for the column - record id indexes to the individual pdb record the data is associated with - entry id each line of the list has a unique id - value stored as a string - timestamp for when the value was last changed this way, we don't need to maintain the table as columns are changed.
