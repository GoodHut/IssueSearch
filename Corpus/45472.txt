 URL : "https://github.com/olliebudge/f1-3-c2p1-colmar-academy/issues/2" TITLE : ids have to be unique BODY : you use id= dt multiple times. this breaks the html 5 standard. classes are reusable, ids have to be unique. https://html.spec.whatwg.org/multipage/dom.html the-id-attribute two main reasons for this rule are: first ids are used for in page linking. if you use multiple ids with the same name the link will go to the nearest one. second any scripting language that uses ids as an identifier which is nearly all of them. biggest example here is jquery, when it is used to add an element based on a page event, it expects an unique id otherwise it will break the script and the website.
