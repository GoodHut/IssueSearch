 URL : "https://github.com/bebuch/disposer/issues/4" TITLE : inline module definition BODY : currently we have a module section which does map a module type name to a free definable module name and which defines the module parameters. in the chain section we refer to the defined module name. module module_name = module_type_name param1 = value chain chain_name other modules … module_name <- input1 = input_variable -> output1 = output_variable this should be changed to a inline definition within the chains: chain chain_name other modules … module_type_name parameter param1 = value <- input1 = input_variable -> output1 = output_variable so we don't need module_name's anymore and have a single view over all important module stuff. parameters, inputs, outputs the parameter_set's are redundant to the previous module section and can completely replace this indirection layer.
