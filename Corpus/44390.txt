 URL : "https://github.com/tc39/proposal-realms/issues/45" TITLE : evaluator for modules BODY : new method realm.prototype.evalmodule which will be equivalent to realm.prototype.eval , but evaluates a module source text. this new method should be promise based, returning a promise to a module namespace object: js const r = new realm ; const ns = await r.evalmodule export const x = 1; ; console.log ns.x ; // yield 1 this new method is just a convenience since you can achieve similar results by using the new import syntax: js const r = new realm ; const ns = await r.eval import something ; console.log ns.x ; // yield 1 where something resolves to the source text export const x = 1; . additionally, we can bikeshed on the names, and get realm.prototype.eval to align with the new method. @dherman suggested: realm.prototype.evalscript scripttext and realm.prototype.evalmodule sourcetext or just realm.prototype.script scripttext and realm.prototype.module sourcetext .
