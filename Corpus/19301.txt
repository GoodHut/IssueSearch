 URL : "https://github.com/SO-II-2017/shelly/issues/31" TITLE : 6 kill the zombies BODY : run the command $ echo x | cat | cat | ./busy the pipeline shall stuck, as busy does nothing but pause. when all previous commands in the pipeline have been finished, they must be removed by the kernel from the list of process. in order to verify if this is happening, in another terminal, with bash, run the command $ ps uf you should not see zombie cats. if you do, it is because your shell is not calling syscall wait for the cat commands when they terminate. check your code. tip: set up a handler for sigchld signal where you go thought list of child process, calling waitpid in nonblocking mode.
