 URL : "https://github.com/MOLB7621/Discussion/issues/37" TITLE : bedtools groupby bug BODY : there is a bug in bedtools version 2.26.0 that is causing the error with groupby see arq5x/bedtools2 484 . if you have version 2.26.0 then you will need to use a previous version. we can help you install the older version in class next week. however if you'd like to try to install an older version yourself, execute the following commands. if you are working on tesla your bedtools version does not need to be changed. bash check your version bedtools --version uninstall bedtools via homebrew if you are on a mac brew uninstall bedtools download bedtools wget https://github.com/arq5x/bedtools2/releases/download/v2.25.0/bedtools-2.25.0.tar.gz decompress tarball tar -zxvf bedtools-2.25.0.tar.gz compile c++ code cd bedtools2 make copy binaries to a location in your path cp bin/ /usr/local/bin you may need to use sudo sudo cp bin/ /usr/local/bin check if version has changed to 2.25.0 bedtools --version if you get any error messages after you run make post your output here. warnings are ok. if you are on linux then the install procedure is similar additional instructions here http://bedtools.readthedocs.io/en/latest/content/installation.html installing-stable-releases , but you need to make sure that the new binaries are used instead of the old binaries.
