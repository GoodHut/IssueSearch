 URL : "https://github.com/kubernetes/kubernetes/issues/44095" TITLE : infrastore for kubernetes BODY : internally at google infrastore is a repository of data about google infrastructure, which drastically improved debugging experience for borg users and became a foundation of a number of systems like autopilot or sigma. it was described in borg paper https://static.googleusercontent.com/media/research.google.com/pl//pubs/archive/43438.pdf : borg records all job submissions and task events, as well as detailed per-task resource usage information in infrastore, a scalable read-only data store with an interactive sql-like interface via dremel 61 . this data is used for usage-based charging, debugging job and system failures, and long-term capacity planning. it also provided the data for the google cluster workload trace 80 . we want to build a similar system for kubernetes project.
