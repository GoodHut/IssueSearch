 URL : "https://github.com/iGenius-Srl/realmGenerator/issues/1" TITLE : support for primary keys BODY : actually, the realm generator does not have anything to be able to annotate a field as a primary key. given a json model: json { user : { id : asdsadsadsadzc , name : john , surname : smith , email : test@igenius.net , image : http://localhost:8080/img/user.png , } } i want to mark the field id as primary key. this can be accomplished by putting an @ in front of id , like this: json { user : { @id : asdsadsadsadzc , name : john , surname : smith , email : test@igenius.net , image : http://localhost:8080/img/user.png , } } this will result in the following code getting generated: java java public class user extends realmobject { public static final string id = id ; public static final string name = name ; public static final string surname = surname ; public static final string email = email ; public static final string image = image ; @primarykey private string id; private string name; private string surname; private string email; private string image; // getters and setters } swift swift class user: object { dynamic var id = dynamic var name = dynamic var surname = dynamic var email = dynamic var image = override static func primarykey -> string? { return id } }
