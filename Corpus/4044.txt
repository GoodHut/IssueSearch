 URL : "https://github.com/spinnaker/spinnaker/issues/2085" TITLE : add fail stage BODY : feature area pipelines description in our deployment pipeline, we run a smoke test after a deployment. if the smoke test fails, we rollback to the previous cluster and destroy the newly created cluster. in order to do that, we don't fail the pipeline after a smoke failure. depending on the result we either complete the deployment or rollback to the previous cluster. in the failure case, after rolling back we want to fail the pipeline. at the moment, as a workaround, we use check preconditions stage with the expression set to false but that is not ideal. we think that having a fail stage will be a better approach.
