 URL : "https://github.com/ConfigHubPub/ConfigHubPlatform/issues/2" TITLE : type awareness for files BODY : currently, we're able to define files using any format that we wish. this is a great property; however, it opens up the possibility of defining a file that doesn't match its expected format. it would be nice to be able to define the intended type of a file within reason, something like application/json , application/xml , etc. and have confighub validate that the file definition is correct before allowing changes to be saved.
