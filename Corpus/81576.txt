 URL : "https://github.com/kraih/mojo-pg/issues/44" TITLE : ->reduce examples broken? BODY : mojo::pg version: 4.01 perl version: any operating system: any this is a documentation bug. the pod for mojo::pg::results contains these examples: say $results->arrays->reduce sub { $a-> 3 + $b-> 3 } ; say $results->hashes->reduce sub { $a->{money} + $b->{money} } ; which fail to work if there are more than two elements in $results . the proper code for the second example would be: say $results->hashes->reduce sub { { money => $a->{money} + $b->{money} } } ->{money};
