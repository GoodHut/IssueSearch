 URL : "https://github.com/AutolabJS/AutolabJS/issues/174" TITLE : hot update containers BODY : the autolab containers are heavy duty and can't be easily reinstalled. it is better to hot update the containers with the latest code and restart them. there are two approaches for this. 1. map volumes to containers main server, load balancer and execution nodes copy mysql, gitlab only map volumes from host. if we follow similar approach and map the code base, it becomes easier to update. we only have to stop the container, update the base code directory and restart the container. 2. copy tar files, untar inside copy the update tar bundle into container and untar it inside. that would automatically update the contents. the hot update reduces the deployment time and is very useful for developers.
