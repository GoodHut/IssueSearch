 URL : "https://github.com/NetLogo/NetLogo/issues/1342" TITLE : reduce documentation bug BODY : version 6.0 documentation for reduce has this example reduce result-so-far next-item -> fput result-so-far next-item fput 1 2 3 4 5 this generates a runtime error fput expected input to be a list but got the number 1 instead. referring to the first fput. the corrected example should be: reduce result-so-far next-item -> fput next-item result-so-far fput 1 2 3 4 5 => 5 4 3 2 1 compare the code in version 5.3.1 show reduce fput ?2 ?1 fput 1 2 3 4 5 => 5 4 3 2 1
