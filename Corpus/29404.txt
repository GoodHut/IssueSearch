 URL : "https://github.com/gfx-rs/gfx/issues/1553" TITLE : ll adapter features BODY : vulkan requires enabling physical device / adapter features when creating a device e.g depth clamping . easiest option would probably to query all features and enabling them, but this might come with some performance cost. for the sake of vulkan portability we might just follow vulkan here and force the user to set all wanted features at adapter::open . relevant vulkan specs section: https://www.khronos.org/registry/vulkan/specs/1.0/html/vkspec.html devsandqueues-physical-device-enumeration
