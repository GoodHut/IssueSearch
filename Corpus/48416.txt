 URL : "https://github.com/boto/boto3/issues/1171" TITLE : s3 tagset loses order BODY : versions: boto3: 1.4.4 python: 3.5.3+ issue: the order of tags in the 'tagset' list returned by s3.client.get_bucket_tagging seems to be different than the order of tags included in 'tagset' for s3.client.put_bucket_tagging. this new order that is returned is neither reversed nor alphabetic by keys but it is consistently the same different order across multiple calls with the same list of tags. example: client.put_bucket_tagging bucket=bucket_name, tagging={ 'tagset': {'key': 'env', 'value': 'test'}, {'key': 'builddate', 'value': 'now'}, {'key': 'chance', 'value': 'go'}, {'key': 'date', 'value': 'then'}, {'key': 'first', 'value': '1'}, {'key': 'second', 'value': '2'} } response = client.get_bucket_tagging bucket=bucket_name print response 'tagset' > {'value': '2', 'key': 'second'}, {'value': 'now', 'key': 'builddate'}, {'value': '1', 'key': 'first'}, {'value': 'test', 'key': 'env'}, {'value': 'then', 'key': 'date'}, {'value': 'go', 'key': 'chance'} is there a reason why the list is losing its order?
