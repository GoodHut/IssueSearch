 URL : "https://github.com/kurttheviking/git-rev-sync-js/issues/27" TITLE : add --first-parent to tag BODY : hi @kurttheviking , please add the option --first-parent to gitrevsync.tag describe command. option details is here https://git-scm.com/docs/git-describe this option will return the tag which originated on the current branch, currently if i make a hotfix in the master and merge backwards in develop the hotfix tag will be returned by gitrevsync.tag instead of the dev tag. example: develop: 1.3.0-dev master: 1.2.1 after merge from master to develop. on develop branch the tag will be 1.2.1
