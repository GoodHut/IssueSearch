 URL : "https://github.com/sacmwg/draft-ietf-sacm-terminology/issues/41" TITLE : thoughts on data origin BODY : the way our definition for _data origin_ is written makes it seem like the origin of data is a set of endpoint attributes while creating a synonym for endpoint attributes in properties . this does not feel accurate. the _data origin_ is _the sacm component that initially acquired or produced data about an endpoint_. we may call that sacm component the _originating sacm component_. then we an use the last sentence of the existing definition: _data origin can be expressed by an endpoint label information element_. although, that feels more complicated than is necessary and we may instead want to say something like: _data origin is represented by one or more endpoint attributes intended to identify the originating sacm component_.
