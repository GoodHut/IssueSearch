 URL : "https://github.com/openshmem-org/specification/issues/193" TITLE : lock api and contexts BODY : issue openshmem 1.4 specifies: > the shmem_clear_lock routine releases a lock previously set by shmem_set_lock after ensuring that all local and remote stores initiated in the critical region are complete. the remote stores bit needs clarification. in particular, does this requires that all contexts touched during the critical section must be quieted? what if the context is private or serialized? proposed solution > the shmem_clear_lock routine releases a lock previously set by shmem_set_lock after ensuring that all local stores and _operations initiated on the default context_ in the critical region are complete.
