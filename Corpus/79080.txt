 URL : "https://github.com/zero-os/0-stor/issues/33" TITLE : review reservation logic BODY : the current implementation of the reservations and how the token are generated is wrong. this parts needs to be re-written. concepts: at the creation of a reservation, 2 token are created - reservationtoken : this is used to managed the reservation. it's needed if user want to renew a reservation - dataaccesstoken: this token is needed to access the reservation and read/write object on the store. multiple dataaccesstoken can be created for different users. both of these token are jwt, the current implementation doesn't generate jwt. requirements: - the store needs to generate a key pair that is going to b used to sign the jwt. - token generation needs to be updated to generate jwt
