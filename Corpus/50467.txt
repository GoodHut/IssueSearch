 URL : "https://github.com/madrobby/zepto/issues/1261" TITLE : .attr changed in 1.2.0 BODY : i just upgrade a project to 1.2.0 and caught a bug. turns out .attr changed it's return value when the attribute doesn't exist: // 1.1.7 $ 'body' .attr 'non-existing' ; // null // 1.2.0 $ 'body' .attr 'non-existing' ; // undefined see: 1.2.0: http://jsbin.com/tivozelili/edit?html,js,console,output 1.1.7: http://jsbin.com/nosafoniyu/1/edit?html,js,console,output i guess this is as it should be when i read 148 and the applied fix. that only fixed the return value of attr on an empty selector, not with a valid selector but not existing attribute. could this be added to the changelog of 1.2.0? this might break more apps.
