 URL : "https://github.com/Clarifai/clarifai-javascript/issues/14" TITLE : node dependency issue BODY : i had an issue with version 2.1.4dev which is currently the default module on npm . i'm using clarifai with react native; following import, i got this error: unabletoresolveerror: unable to resolve module domain from /users/user/documents/projectname/node_modules/asap/raw.js: module does not exist in the module map or in these directories: /users/user/documents/projectname/node_modules , /users/user/node_modules this might be related to https://github.com/facebook/react-native/issues/4968 to resolve try the following: 1. clear watchman watches: watchman watch-del-all . 2. delete the node_modules folder: rm -rf node_modules && npm install . 3. reset packager cache: rm -fr $tmpdir/react- or npm start --reset-cache the problem here is domain has been deprecated in node.js https://nodejs.org/api/domain.html domain_domain but is used in asap@2.0.5 a dependency of promise@7.1.1, which itself is a dependency of clarifai @2.1.4-dev . i was able to get around this issue by using clarifai@2.0.9, which uses a different promise library i'm using yarn, the command was: yarn add clarifai@2.0.9 . here's a screenshot of the old dependency tree ! screen shot 2017-03-21 at 3 14 04 pm https://cloud.githubusercontent.com/assets/21132299/24200550/d30d2ce0-0ee3-11e7-8ae3-f54d2ffde0f7.png and the new dependency tree: ! screen shot 2017-03-22 at 9 58 26 am https://cloud.githubusercontent.com/assets/21132299/24201362/37ff2c0a-0ee6-11e7-8c3e-bbfef63a80b7.png summary: most recent clarifai release uses a library which uses a deprecated node module
