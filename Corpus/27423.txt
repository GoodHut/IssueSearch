 URL : "https://github.com/konigio/konig/issues/432" TITLE : data access servlet BODY : i want a servlet that will handle requests to access instances of a given data shape acceptance criteria - must implement a container for each owl class. - the url for the container must have the form {baseurl}/{classid} where classid is a slug that is mapped to the owl class iri via a configuration file. - the servlet must support access to entities of a given class via a url of the form {baseurl}/{classid}?{propertyname} suppose we define person as the classid for the schema:person class. then it should be possible to lookup all people with the family name smith via a query of the form: /person?familyname=smith
