 URL : "https://github.com/contao/contao-manager/issues/61" TITLE : use cookie for jwt token BODY : the current authentication must be replaces. 1. auth is not related to tenside, it should be an application task. 2. we must not use authorization: bearer because. 1. some apache configs block it https://github.com/symfony/symfony/issues/1813 , and we can't include an .htaccess file to fix that. 2. the token needs to be stored by javascript e.g. in localstorage , which means an xss attack can read it. the only correct solution is to use cookies and request tokens: https://stormpath.com/blog/where-to-store-your-jwts-cookies-vs-html5-web-storage where-to-store-your-jwts it also means we automatically don't need to handle storing and lifetime of the token.
