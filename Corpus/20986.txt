 URL : "https://github.com/wso2/siddhi/issues/386" TITLE : disk backed window BODY : reporter : manoj gunawardena manojg@wso2.com jirra url : https://wso2.org/jira/browse/siddhi-4 url description : for large time frames and large number of events the memory based event queues might be running out of memory. the idea of disk backed window is, when the number of events increased in the incoming queue, the part of the queue is written to the disk and once events required to emit, read from disk and load to memory in advance.
