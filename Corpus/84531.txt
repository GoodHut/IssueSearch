 URL : "https://github.com/volkszaehler/volkszaehler.org/issues/572" TITLE : debug-feature im frontend BODY : für debug-zwecke, export von raw-zählerständen, tests u.ä. folgende idee: es wäre doch sehr benutzerfreundlich, wenn man im frontend ein fensterchen öffnen könnte, wo man per üblichem dropdown die passende middleware wählen und darunter in ein einfaches textfeld seinen request ab ? eingeben kann. klick auf ok verpackt das ganze richtig und lädt die antwort in neuem tab/als file. besonders interessant auch im hinblick auf pr 551 access control , denn eine installation mit aktiver firewall akzeptiert keine requests an die middleware, weil das authtoken nicht übergeben wird. da das frontend die ganze infrastruktur für login und token schon hat, sollte es keine problem sein, das token in den header des gewünschten requests einzubauen.
