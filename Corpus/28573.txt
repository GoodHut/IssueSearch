 URL : "https://github.com/oncletom/tld.js/issues/104" TITLE : handle ip addresses BODY : hi, currently we don't detect if a url contains a hostname or an ip address. we should probably add some check so that we return a consistent result. current behavior: js > tldjs.parse 'https://123.23.32.31' { valid: true, hostname: '123.23.32.31', suffix: '31', domain: '32.31', subdomain: '123.23' } should probably be something like: js > tldjs.parse 'https://123.23.32.31' { valid: true, hostname: '123.23.32.31', suffix: null, domain: null, // or hostname ? subdomain: null }
