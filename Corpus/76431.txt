 URL : "https://github.com/juju/docs/issues/1664" TITLE : juju gui command changes BODY : the juju gui command has changed. no-browser is now the default behaviour. it also prints the login credential by default. there is a new --hide-credential option not to show the credential. the --no-browser option is supported but deprecated it is effectively a no-op . to bring up a browser, use the --browser option. snippet from juju gui --help details: print the juju gui url and show admin credential to use to log into it: juju gui print the juju gui url only: juju gui --hide-credential open the juju gui in the default browser and show admin credential to use to log into it: juju gui --browser open the juju gui in the default browser without printing the login credential: juju gui --hide-credential --browser
