 URL : "https://github.com/brunchboy/dysentery/issues/7" TITLE : fader start packet BODY : i didn't see anything about fader start in the analysis, but i found the actual packet in libpdjl https://bitbucket.org/awwright/libpdjl . without having to announce a mixer on the network, i was able to get an xdj-1000mk1 to start playing or jump back to the active cue, just by broadcasting the following packet on port 50001: 51 73 70 74 31 57 6d 4a 4f 4c 02 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 d 00 04 f1 f2 f3 f4 where d is the mixer's channel 0x21 , and f1 to f4 are the fader start values for each player: | value | action | | --- | --- | | 0x00 | play | | 0x01 | cue | | 0x02 | none | it would be great to have a virtualdjm class in beat-link to be able to send these packets, for the people that don't have a djm with pro dj link.
