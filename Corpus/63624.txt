 URL : "https://github.com/ember-decorators/argument/issues/30" TITLE : enum / values validator BODY : i often write components that take specific string values as configuration parameters. for example, an icon component might take 4 different, pre-defined, named sizes, like small , medium , large , extra-large . i'd like a validator that only allows these specific values, for instance: js import component from '@ember/component'; import { argument } from '@ember-decorators/argument'; import { values } from '@ember-decorators/argument/type'; import { immutable } from '@ember-decorators/argument/validation'; export default class iconcomponent extends component { @argument @values 'small', 'medium', 'large', 'extra-large' @immutable size = 'medium'; } i am open to name suggestions!
