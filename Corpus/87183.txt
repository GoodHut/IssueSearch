 URL : "https://github.com/uikit/uikit/issues/2830" TITLE : issue with svg-fill mixin BODY : uikit version 3.0.0-beta.28 browser microsoft edge 40.15063.0.0, edgehtml 15.15063 steps to reproduce pass a hsl or hsla string as $color-new param to the svg-fill mixin. what is expected? a valid encoded base64 string is returned as background image. what is actually happening? no background image is visible in microsoft edge windows 10 because some characters of the hsl or hsla string aren't replaced. this may also happend in older browser versions with rgb or rgba values. works fine on macos 10.12.6 : apple safari 10.1.2 google chrome 60.0 mozilla firefox 54.0.1 works fine on windows 10: google chrome 60.0 mozilla firefox 54.0.1 possible fix @mixin svg-fill $src, $color-default, $color-new { $color-new: quote $color-new ; $color-new: str-replace $color-new, , ; // replace spaces $color-new: str-replace $color-new, % , %25 ; // replace % of hsl/hsla color values $color-new: str-replace $color-new, , , %2c ; // replace commas $color-new: str-replace $color-new, , %28 ; // replace opening parentheses $color-new: str-replace $color-new, , %29 ; // replace closing parentheses $color-new: unquote $color-new ; $replace-src: str-replace $src, $color-default, $color-new !default; $replace-src: str-replace $replace-src, , %23 ; background-image: url quote $replace-src ; } example calls @include svg-fill $internal-accordion-close-image, 000 , 00ff00 ; @include svg-fill $internal-accordion-close-image, 000 , rgb 0, 255, 0 ; @include svg-fill $internal-accordion-close-image, 000 , rgba 0, 255, 0, 0.75 ; @include svg-fill $internal-accordion-close-image, 000 , hsl 120, 100%, 50% ; @include svg-fill $internal-accordion-close-image, 000 , hsla 120, 100%, 50%, 0.75 ;
