 URL : "https://github.com/bolt-project/bolt/issues/103" TITLE : type of elements in shape BODY : usually, if a is a boltarray , then a.shape returns a list with members of type int . however, after chunking and then calling either keys_to_values or values_to_keys , the list returned by a.shape have members of type numpy.int64 . this inconsistency has be a problem if a user tries to serialize this data such as in thunder, when writing out the shape as part of metadata , as python's json package does not know how to handle this case.
