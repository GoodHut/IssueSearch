 URL : "https://github.com/tarantool/tarantool/issues/2355" TITLE : views in sql BODY : some problems with views in sql. a view is not dropped when its underlying table is dropped. this is not uncommon, other dbmss also allow out-of-date views. but it is not helpful, either. box.sql.execute create table te22 s1 int primary key ; box.sql.execute create view te23 as select from te22; box.sql.execute drop table te22; box.sql.execute select from te23; result = error: no such table: main.te22 it is not possible to insert or update or delete from a view, although sometimes a possible substitution is to create an instead of trigger.
